סביר להניח שאתם כבר משתמשים ב-AI לפיננסים אישיים בצורה כלשהי. אולי אתם מדביקים דף חיוב בנקאי לתוך ChatGPT ומבקשים ממנו לקטלג את ההוצאות שלכם. או מצלמים את אפליקציית הבנק ומבקשים מ-Claude לחשב כמה הוצאתם על קניות החודש.

זה עובד פעם אחת. אבל התשובה נשארת בצ'אט. שום דבר לא נשמר, שום דבר לא מנוהל, ושבוע אחר כך אתם עושים בדיוק את אותו דבר שוב. ה-AI קורא את הנתונים שלכם, נותן סיכום, ואז הכול נעלם.

יש דרך שימושית יותר להשתמש ב-AI למעקב הוצאות. במקום לבקש מה-AI לנתח צילומי מסך, תנו לו גישת כתיבה אמיתית למסד הנתונים הפיננסי שלכם. תנו ל-AI לרשום טרנזקציות, לעדכן את התקציב ולאמת יתרות ישירות, לא רק לדבר עליהן.

## איך "מעקב הוצאות עם AI" נראה בפועל

Kirill Markin, היוצר של [Expense Budget Tracker](https://expense-budget-tracker.com/he/), מקטלג כל טרנזקציה אישית כבר יותר מחמש שנים. הוא התחיל לעשות את זה ידנית, ואז התחיל לבנות כלים שיהפכו את זה למהיר יותר. המערכת הנוכחית משתמשת בסוכן AI שמתחבר ישירות למסד הנתונים דרך SQL API.

שגרת השבוע שלו נראית כך: מורידים דפי חיוב בנקאיים כ-CSV או PDF, זורקים אותם לתוך סוכן AI, נותנים לו לנתח כל טרנזקציה ולרשום אותה. הסוכן כבר מכיר את קטגוריות ההוצאה שלו מרשומות קודמות, ולכן הוא מתאים את רוב הטרנזקציות נכון בעצמו. Kirill עובר על מה שה-AI עשה, מתקן את מעט הטעויות, וממשיך הלאה. כל התהליך לוקח בערך 10 דקות, במקום שעה כשהוא היה מזין הכול ידנית.

אותה גישה עובדת עם [Claude Code](https://docs.anthropic.com/en/docs/claude-code), עם [OpenAI Codex](https://openai.com/index/codex/), עם GPTs מותאמים אישית, או עם כל סוכן AI שיכול לקרוא לנקודות קצה ב-HTTP. המרכיב המרכזי הוא גישה ישירה למסד הנתונים, לא plugin, לא תוסף דפדפן, אלא מפתח API שמאפשר ל-AI לקרוא ולכתוב את הנתונים הפיננסיים שלכם.

## איך לחבר את סוכן ה-AI שלכם לפיננסים שלכם

[Expense Budget Tracker](https://expense-budget-tracker.com/he/) הוא מערכת פיננסים אישיים בקוד פתוח שמבוססת על Postgres. יש לה נקודת קצה של SQL API ב-`POST /v1/sql` שמקבלת שאילתות SQL דרך HTTP ומחזירה תוצאות JSON.

כדי לחבר כל סוכן AI:

1. פתחו את האפליקציה ועברו אל **Settings → API Keys → Create key**
2. העתיקו את המפתח, הוא מתחיל ב-`ebt_` ותראו אותו פעם אחת בלבד
3. ספרו לסוכן ה-AI שלכם שני דברים: כתובת נקודת הקצה של ה-API ואת המפתח

זה הכול. הסוכן יכול עכשיו לשאול ולשנות את נתוני ההוצאות שלכם. בלי שרת MCP שצריך להחזיק חי, בלי plugin שצריך להתקין, ובלי אינטגרציה מותאמת שצריך לתחזק. כל AI שיודע לבצע HTTP POST, ובעצם כולם יודעים, עובד ישר מהקופסה.

```bash
curl -X POST https://api.expense-budget-tracker.com/v1/sql \
  -H "Authorization: ApiKey ebta_your_key_here" \
  -H "X-Workspace-Id: workspace-id" \
  -H "Content-Type: application/json" \
  -d '{"sql": "SELECT * FROM ledger_entries ORDER BY ts DESC LIMIT 10"}'
```

התגובה חוזרת כמערך JSON של שורות. בלי pagination tokens, בלי אובייקטים מקוננים, ובלי SDK.

## מה סוכן ה-AI שלכם יכול לעשות עם הגישה הזאת

עם מפתח ה-API, סוכן ה-AI עובד על נתוני ההוצאות והתקציב האמיתיים שלכם, לא על עותק, לא על סיכום, אלא על מסד הנתונים החי:

**לנתח ולרשום הוצאות.** זרקו דף חיוב בנקאי, CSV, PDF או צילום מסך של אפליקציית הבנק, לתוך סוכן ה-AI שלכם. הסוכן קורא כל שורה, מבין את הסכום, התאריך, הצד שכנגד והקטגוריה, ואז כותב פקודת `INSERT` לטבלת `ledger_entries`. כל הוצאה נכנסת ישירות למסד הנתונים.

**לסווג טרנזקציות בעזרת הקטגוריות הקיימות שלכם.** הסוכן מתחיל מלשאול באילו קטגוריות אתם כבר משתמשים. אם אתם מסווגים את "Whole Foods" כ-"groceries" כבר חודשים, ה-AI לומד את זה ועושה אותו דבר. אתם לא צריכים להסביר מחדש את המערכת שלכם בכל פעם.

**לבדוק יתרות חשבון.** אחרי רישום כל ההוצאות מדף חיוב בנקאי, הסוכן יכול לשאול את תצוגת `accounts` ולהשוות את הסכומים מול מה שמופיע בבנק. אם משהו לא מסתדר, אתם יודעים שחסרה טרנזקציה.

**לעדכן את תחזית התקציב שלכם.** הטבלה `budget_lines` מחזיקה את תוכנית התקציב החודשית, הכנסה צפויה והוצאות מתוכננות לפי קטגוריה. סוכן ה-AI יכול לקרוא את הביצוע של החודש, להשוות לתוכנית, ולהציע או לבצע ישירות התאמות לחודש הבא.

**לעבוד עם כמה מטבעות.** כל טרנזקציה במסד הנתונים נשארת במטבע המקורי שלה. שערי חליפין נמשכים מדי יום מ-ECB, מ-CBR ומ-NBS. ה-AI לא צריך להמיר שום דבר. מסד הנתונים עושה את מתמטיקת המטבע בזמן השאילתה.

הסכמה מכוונת להיות שטוחה ופשוטה. שבע טבלאות, שמות עמודות ברורים, בלי מבנים מקוננים עמוקים. מודלי AI מייצרים שאילתות SQL נכונות מול הסכמה הזו כבר בניסיון הראשון, כי כמעט אין בה מה להבין לא נכון.

## צ'אט AI מובנה למשימות מהירות

ל-Expense Budget Tracker יש גם עוזר AI מובנה בתוך ממשק האינטרנט. אתם מחברים מפתח OpenAI או Anthropic ב-Settings, והצ'אט מקבל כלי `query_database`, אותה גישת SQL אבל מתוך האפליקציה עצמה.

זה נוח למשימות מהירות: מעלים צילום מסך של קבלה, מבקשים מה-AI להוסיף אותה כהוצאה, מאשרים, וסיימתם. ה-AI המובנה עובד לפי פרוטוקול קפדני: הוא בודק את הקטגוריות הקיימות, מחפש כפילויות, מאמת יתרות, וכותב למסד הנתונים רק אחרי אישור שלכם.

למשימות גדולות יותר, עיבוד באצ'י של כמה דפי חיוב, בניית תהליכי עבודה אוטומטיים או אינטגרציה עם מערכות אחרות, ה-SQL API החיצוני מעשי יותר. אפשר להשתמש בו מכל סוכן או סקריפט מחוץ לאפליקציה.

## למה SQL ישיר עדיף על שרתי MCP ו-plugins

שרתי MCP, פעולות מותאמות ב-GPTs ו-plugins תלויי ספק פופולריים כרגע לחיבור AI לכלים חיצוניים. בפיננסים אישיים הם מוסיפים חלקים נעים שלא באמת צריך.

שרת MCP הוא תהליך נוסף שצריך להחזיק חי. אם הוא קורס, ה-AI מאבד גישה לנתוני ההוצאות שלכם באמצע שיחה. plugins מותאמים ל-GPT עובדים רק עם ChatGPT, ולא יעזרו אם תעברו ל-Claude או תבנו סוכן משלכם. אינטגרציות תלויות ספק נשברות בכל פעם שהספק מעדכן API.

SQL API עוקף את כל זה. הממשק הוא נקודת קצה ב-HTTP ושפת SQL. שניהם קיימים כבר עשרות שנים ולא הולכים לשום מקום. תעברו ממודל AI אחד לאחר, אותו מפתח API, אותה נקודת קצה, אותו SQL. לסוכן ה-AI לא אכפת אם הוא רץ בתוך ChatGPT, ב-Claude Code או בסקריפט Python שכתבתם בעצמכם.

## האם בטוח לתת ל-AI גישה ישירה למסד נתונים?

כן, במסגרת המגבלות הנכונות. SQL API של Expense Budget Tracker אוכף כמה שכבות הגנה:

כל שאילתה רצה דרך Postgres Row Level Security. מפתח ה-API קשור למשתמש ולסביבת העבודה שלכם, ולכן ה-AI יכול לראות ולשנות רק את נתוני ההוצאות שלכם, לא של אף אחד אחר.

מותרות רק פעולות נתונים: `SELECT`, `INSERT`, `UPDATE`, `DELETE`. סוכן ה-AI לא יכול ליצור טבלאות, למחוק סכמות או לשנות הרשאות. כמה פקודות בבקשה אחת חסומות. גם `set_config()` חסום, כדי למנוע הסלמת הרשאות.

מפתחות API נשמרים כ-hash של SHA-256, כך שהטקסט הגלוי לא יושב אף פעם במסד הנתונים. אפשר לבטל מפתח מיידית מתוך Settings. אם מסירים חבר מסביבת עבודה, כל המפתחות שלו נמחקים אוטומטית.

מגבלות קצב עוצרות שימוש ב-10 בקשות לשנייה ו-10,000 ביום לכל מפתח. שאילתות מקבלות timeout אחרי 30 שניות. תגובות מחזירות עד 100 שורות. המספרים האלה מספיקים בהחלט למעקב הוצאות ותקציב עם AI, ובו-בזמן מונעים התנהגות פראית.

## טיפים מעשיים למעקב הוצאות עם AI

כמה דברים שהופכים את תהליך העבודה של מעקב הוצאות עם AI לחלק יותר, מתוך שימוש יומיומי אמיתי:

**שמרו על עקביות בקטגוריות ההוצאה שלכם.** ה-AI לומד מהנתונים שכבר קיימים. אם לפעמים אתם קוראים לקטגוריה "restaurants" ולפעמים "dining out", הסוכן יתבלבל. בחרו שם אחד לכל קטגוריה והישארו איתו.

**אמתו יתרות כל שבוע.** אחרי שה-AI רושם הוצאות מדף חיוב בנקאי, בדקו שיתרת החשבון במערכת תואמת למה שמופיע בבנק. כך תופסים מוקדם טרנזקציות חסרות או כפולות, לפני שהן מצטברות.

**התחילו עם חשבון אחד.** אל תנסו להגדיר ביום הראשון את כל חשבונות הבנק, כרטיסי האשראי וחשבונות ההשקעה שלכם. התחילו מחשבון העו"ש הראשי. תנו ל-AI לטפל בו כמה שבועות. הוסיפו עוד חשבונות ברגע שתהליך העבודה מרגיש יציב.

**עברו על הקטלוג של ה-AI בכל פעם.** ה-AI מסווג נכון את רוב הטרנזקציות, אבל מדי פעם יטעה, במיוחד עם בתי עסק חדשים או הוצאות חריגות. השקיעו חמש דקות בבדיקה. תיקון הטעויות שלו משפר את הדיוק העתידי, כי בפעם הבאה הוא לומד מהנתונים המתוקנים.

**השתמשו גם בטבלת התקציב, לא רק במעקב ההוצאות.** לרשום מה שכבר הוצאתם זה שימושי אבל מוגבל. הערך האמיתי הוא בתחזית מתגלגלת ל-12 חודשים, שורות הן קטגוריות, עמודות הן חודשים, חודשים עתידיים מכילים תחזית. סוכני AI טובים בעדכון התחזיות האלה לפי דפוסי ההוצאה בפועל. בקשו מהסוכן לעדכן את תקציב החודש הבא אחרי שבדקתם את החודש הנוכחי.

## איך מתחילים

1. הירשמו ב-[expense-budget-tracker.com](https://expense-budget-tracker.com/he/) או [ארחו בעצמכם](https://github.com/kirill-markin/expense-budget-tracker) את האפליקציה על השרת שלכם
2. עברו אל **Settings → API Keys → Create key** והעתיקו את המפתח
3. תנו לסוכן ה-AI שלכם את המפתח, את מזהה סביבת העבודה ואת נקודת הקצה `https://api.expense-budget-tracker.com/v1/sql`
4. זרקו לסוכן דף חיוב בנקאי ובקשו ממנו לנתח ולרשום את ההוצאות

ה-AI יגלה את סכמת מסד הנתונים, יתאים את קטגוריות ההוצאה שלכם ויתחיל לכתוב טרנזקציות. עברו על מה שנרשם, תקנו כל דבר חריג, ויש לכם תקציב מנוהל ב-AI שעובד.

Kirill Markin כתב בפירוט על השיטה האישית שלו כאן: [How I Use AI to Handle My Expenses from Bank Accounts and Budget](https://kirill-markin.com/articles/ai-expense-tracking-bank-accounts-budget/). חמש שנים של כל טרנזקציה מקוטלגת ומנוהלת, בדיוק לפי אותה גישה שמתוארת במאמר הזה, ונבדקה על כסף אמיתי בכמה מטבעות ובכמה מדינות.

הכלי מופץ ברישיון MIT והוא בקוד פתוח מלא ב-[github.com/kirill-markin/expense-budget-tracker](https://github.com/kirill-markin/expense-budget-tracker). אפשר להשתמש בגרסה המתארחת או להריץ אותו בעצמכם, וה-SQL API עובד אותו דבר בשני המקרים.
